# DevSecOps Delivery Guide

This folder brings security into the delivery path.
It should feel like the controls a real platform team puts in place so insecure code never gets too far.

## What This Folder Covers
- Shift-left security
- IaC scanning
- Container scanning
- Secret detection
- Security gates in pipelines

## Real-World Lens
- Security is not a final review step.
- It is part of the release path, and good controls block issues before they become incidents.

## Why It Matters
- DevSecOps protects the platform and the people operating it.
- The best security check is the one that stops bad change before it ships.
