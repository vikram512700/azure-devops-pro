# Docs Hub

This is the front door to the learning library.

If the platform is the product, the docs are the field manual.
Every folder is written to feel like a real engineer explaining what they built, what broke, how they fixed it, and what it proves in an interview.

## How to Use This Hub
- Start with the technology you are actively learning.
- Jump to the troubleshooting sections when you want a runbook-style answer.
- Use the project ideas to translate knowledge into portfolio work.

## What the Docs Are Trying To Do
- Replace generic textbook tone with real delivery language.
- Tie every concept to enterprise work, release pressure, and production validation.
- Make each chapter useful during learning, debugging, and interviews.

## Folder Guide

| Folder | What It Feels Like |
|---|---|
| `01_Linux` | Server access, recovery, permissions, processes, networking, and on-call basics |
| `02_Scripting_Python` | Small automation tools, parsing, and ops-friendly scripting |
| `03_Git` | Branching, collaboration, release flow, and safe change management |
| `06_Azure` | Azure fundamentals and CLI usage through real platform tasks |
| `07_Terraform` | IaC design, state handling, modules, environments, and production workflows |
| `08_Ansible` | Configuration automation and repeatable server setup |
| `09_CICD` | Pipeline patterns, quality gates, and delivery flow |
| `10_Monitoring` | Metrics, dashboards, alerting, and incident awareness |
| `11_DevSecOps` | Security gates, supply chain checks, and policy thinking |
| `12_Capstone_Projects` | End-to-end project ideas that translate into portfolio proof |

## Experience Standard
- Every chapter should explain the real-world problem.
- Every solution should include validation.
- Every troubleshooting note should feel like something you could use at 2 AM.
- Every project should end with a clear finish line.

## Fast Path
- Need Linux? Go to `01_Linux`.
- Need Azure CLI or Terraform? Go to `06_Azure` and `07_Terraform`.
- Need portfolio proof? Go to `12_Capstone_Projects`.

## What Good Looks Like
- The reader knows what to do next.
- The reader knows why the command matters.
- The reader can explain the result in a hiring conversation.
